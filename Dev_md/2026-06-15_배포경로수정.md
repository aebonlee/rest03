# 2026-06-15 · 배포 경로 수정 (커스텀 도메인 자산 404 해결)

## 증상
- `rest03.dreamitbiz.com`은 200이나 화면이 비어 있음 — JS/CSS가 `/rest03/assets/...` 로 참조되어 **404**.

## 원인
- `vite base: '/rest03/'` (서브패스용)인데, 실제론 **커스텀 도메인 루트(`/`)로 서빙**됨.
- 배포: GitHub Actions(`deploy.yml`, upload-pages-artifact → deploy-pages). 커스텀 도메인 CNAME 파일 없음.

## 조치
- `vite base` → **`/`**.
- `index.html`: favicon `/rest03/favicon.svg` → `/favicon.svg`, OG/Twitter URL·이미지를 `https://rest03.dreamitbiz.com/...` 로 교정.
- `public/CNAME`(`rest03.dreamitbiz.com`) 추가 → 아티팩트에 포함되어 커스텀 도메인 유지.

## 검증
- `npm run build` 후 `dist/index.html` 자산이 `/assets/...`(루트), `dist/CNAME` 포함.
- main push → Actions 자동 배포.
